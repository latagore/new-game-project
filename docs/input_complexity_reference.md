# Input-Complex Games: Reference & Analysis

## Definition
**Input complexity** = games requiring precise timing, sequencing, multi-button coordination, or pattern recognition under time pressure. Success depends on execution skill, not just decision-making.

---

## Rhythm Games

### **Dance Dance Revolution (DDR) / Pump It Up**
- **Input Model**: Step on directional arrows in sync with music scrolling on screen.
- **Complexity**: Multi-limb coordination, stamina, reading ahead, crossovers, holds, jumps (simultaneous arrows).
- **Forgiveness**: Timing windows (Perfect/Great/Good/Miss); slower songs more forgiving.
- **Relevance**: Visual telegraph (scrolling notes), chunked patterns (measures), predictable timing (beat grid).

### **Guitar Hero / Rock Band**
- **Input Model**: Press fret buttons + strum bar in sync with note highway.
- **Complexity**: Chord shapes, hammer-ons/pull-offs (no strum), star power activation mid-song.
- **Forgiveness**: Timing windows, star power as safety net, practice mode with slow-down.
- **Relevance**: Visual anticipation, muscle memory for patterns, performance under pressure.

### **osu!**
- **Input Model**: Click circles, trace sliders, spin spinners with mouse/tablet in rhythm.
- **Complexity**: Cursor precision + timing, reading density, simultaneous hit objects.
- **Forgiveness**: 300/100/50/Miss grades; mods adjust difficulty (Hard Rock, Easy, No Fail).
- **Relevance**: Cursor accuracy under speed, pattern recognition, flow state.

### **Beatmania IIDX / DJ Max**
- **Input Model**: 7-key keyboard + turntable scratch, layered note lanes.
- **Complexity**: Dense polyrhythms, scratch timing, hand independence.
- **Forgiveness**: Gauge systems (survival vs score attack), assist mods.
- **Relevance**: Multi-input streams, reading vertical scrolling, chunked phrases.

### **Crypt of the NecroDancer**
- **Input Model**: Move/attack on the beat in a roguelike dungeon.
- **Complexity**: Rhythm constraint on tactical decisions, enemy patterns sync to beat.
- **Forgiveness**: Bard mode (no rhythm required), training mode, slower tempos.
- **Relevance**: Rhythm + tactics fusion, environmental interaction on beat.

## Fighting Games

### **Street Fighter Series**
- **Input Model**: Directional + button combos (e.g., ↓↘→ + P for hadouken).
- **Complexity**: Frame-perfect links, charge moves, cancels, option selects.
- **Forgiveness**: Input leniency windows (3–5 frames), shortcuts, buffering.
- **Relevance**: Motion inputs, confirm windows, reaction vs prediction.

### **Tekken Series**
- **Input Model**: String-based combos (e.g., 1,2,3 sequences), just-frame inputs.
- **Complexity**: Korean backdash, electric wind god fist (1-frame), juggle timing.
- **Forgiveness**: Buffering, simplified inputs for some moves.
- **Relevance**: Sequenced inputs, timing precision, stance transitions.

### **Super Smash Bros. Melee**
- **Input Model**: Directional + button + timing (tilt vs smash, short hop, wavedash).
- **Complexity**: L-canceling, dash dancing, DI (directional influence), tech skill.
- **Forgiveness**: Minimal (competitive standard); casual play more forgiving.
- **Relevance**: Analog stick precision, multi-input techniques, movement tech.

### **Guilty Gear / BlazBlue (Arc System Works)**
- **Input Model**: Complex motion inputs (632146, 214214), Roman Cancels, gatling chains.
- **Complexity**: Instant air dashes, fast run cancels, tight combo links.
- **Forgiveness**: Stylish mode (auto-combos), input buffer, negative edge.
- **Relevance**: High execution ceiling, resource management (meter), cancel systems.

## Action Games with Complex Inputs

### **Devil May Cry Series (3, 4, 5)**
- **Input Model**: Weapon/style switching mid-combo, directional + button + timing for moves.
- **Complexity**: Jump cancels, guard flying, real-time style switching, enemy step.
- **Forgiveness**: Generous health, checkpoints; mastery for style rank, not survival.
- **Relevance**: Freeform combo expression, resource-less execution, showmanship.

### **Bayonetta Series**
- **Input Model**: Combo strings, dodge offset (hold button during dodge to continue combo).
- **Complexity**: Witch time windows, torture attack QTEs, accessory management.
- **Forgiveness**: Easy/Very Easy auto-mode, generous dodge frames.
- **Relevance**: Timing-based defense (dodge offset), flow interruption recovery.

### **Monster Hunter Series**
- **Input Model**: Weapon-specific movesets with commit animations, directional + button combos.
- **Complexity**: Guard points, animation cancels, item use timing, positioning.
- **Forgiveness**: Healing windows, armor skills reduce punishment, SOS flares.
- **Relevance**: Commitment windows, telegraphed enemy attacks, preparation phase.

### **Nioh / Nioh 2**
- **Input Model**: Stance switching, ki pulse timing (stamina recovery), combo strings.
- **Complexity**: Flux (ki pulse during dodge), yokai abilities, burst counters.
- **Forgiveness**: Checkpoints, summons, over-leveling possible.
- **Relevance**: Stance system (tactical layer), timing-based resource recovery.

### **Sekiro: Shadows Die Twice**
- **Input Model**: Parry timing (deflect), mikiri counter, jump/dodge distinctions.
- **Complexity**: Posture management, prosthetic combos, perilous attack reads.
- **Forgiveness**: Resurrection mechanic, stealth resets; parry window ~6 frames.
- **Relevance**: Reaction-based defense, telegraph reading, rhythm dueling.

- **Complexity**: Hyper dashes, corner boosts, demo dashes (advanced tech).
- **Forgiveness**: Coyote time, jump buffering, assist mode (infinite stamina, invincibility).

### **Super Meat Boy**

- **Complexity**: Momentum conservation, frame-perfect jumps, hazard timing.
- **Forgiveness**: Instant respawn, short levels; no mechanical assists.
- **Relevance**: Trial-and-error iteration, muscle memory, speed optimization.

### **Dustforce**
- **Input Model**: Run, jump, dash, attack (clean dust/enemies) with momentum physics.
- **Complexity**: Ceiling run chains, dash jump angles, combo preservation.
- **Forgiveness**: Replays for study; no mid-level checkpoints.
- **Relevance**: Flow state, momentum puzzles, score attack (SS rank).

## Key Patterns Across Input-Complex Games

### **Forgiveness Mechanisms**
- **Timing windows**: Perfect/Good/Miss grades; buffer systems.
- **Partial success**: Reduced effect instead of total failure.
- **Assist modes**: Slow-down, auto-execute, damage reduction, infinite resources.
- **Practice tools**: Training modes, replays, slow-motion.
- **Checkpoints/Respawns**: Reduce cost of failure iteration.

### **Readability Under Pressure**
- **Visual telegraphs**: Scrolling notes, enemy wind-ups, hazard warnings.
- **Audio cues**: Beat grids, sound effects, callouts.
- **Chunking**: Patterns grouped into phrases/measures/combos.
- **Predictability**: Consistent timing grids, repeating patterns.

### **Skill Curves**
- **Low floor**: Basic success accessible; optional depth for mastery.
- **High ceiling**: Advanced techniques (cancels, tech, optimization) for experts.
- **Mastery expression**: Style ranks, speedruns, score attack, no-hit runs.

### **Input Schemes**
- **Timing-based**: Rhythm games, parries, dodge rolls.
- **Sequenced**: Combo strings, motion inputs, procedures.
- **Simultaneous**: Multi-button chords, multi-limb coordination.
- **Spatial**: Cursor precision, analog stick angles, positioning.
- **Contextual**: Mode switches, stance changes, tool selection.

### **Tension Sources**
- **Time pressure**: Countdowns, increasing speed, enemy aggression.
- **Resource limits**: Stamina, ammo, health, meter.
- **Commitment**: Animation locks, cooldowns, positional risk.
- **Multi-tasking**: Simultaneous objectives, split attention.

## Takeaways for Your Spellcasting Game

### **Borrow These Patterns**
- **Rhythm game telegraphs**: Visual scroll for spell queues or enemy patterns.
- **Fighting game motion inputs**: Directional + button for spell gestures.
- **DMC freeform combos**: Spell chaining with cancel windows, no resource cost for basic casts.
- **Celeste forgiveness**: Coyote time, input buffering, optional assist modes.
- **Hades accessibility**: God mode equivalent (stability decay reduction, wider leniency).
- **Tetris chunking**: Spell patterns grouped into recognizable phrases.
- **Monster Hunter commitment**: Cast animations with weight; preparation stances for safety.
- **Nioh stance system**: Swap between tactical modes (defensive/aggressive spell kits).
- **StarCraft multitasking**: Juggle spell execution + objective timers + positioning.

### **Avoid These Pitfalls**
- **QWOP intentional awkwardness**: Your controls should be challenging but fair, not comedic.
- **Melee execution gate**: Don't require frame-perfect inputs for basic success.
- **Tetris speed wall**: Don't let time pressure become insurmountable; offer scaling difficulty.
- **Fighting game knowledge gate**: Make spell discovery intuitive, not wiki-dependent.

### **Design Principles from Input-Complex Games**
1. **Readable complexity**: Players understand what they need to do; execution is the challenge.
2. **Chunked patterns**: Group inputs into memorable phrases (3–5 inputs max per spell).
3. **Forgiveness layers**: Buffers, leniency, partial success, assists.
4. **Practice space**: Safe environment to learn inputs without pressure.
5. **Mastery expression**: Depth for experts without gatekeeping basic fun.
6. **Visual/audio feedback**: Clear confirmation of successful inputs.
7. **Fail-forward**: Errors create interesting states (miscasts → weaker variants) rather than hard stops.

## Spellcasting Games

### **Magicka Series**
- **Input Model**: Combine up to 5 elements (QWERASDF keys) → cast with mouse click/spray/self-cast.
- **Complexity**: Element interactions (water + lightning = AoE shock; fire + water = steam), friendly fire, rapid sequencing.
- **Forgiveness**: No mana cost; death is frequent but respawn is instant; co-op revives.
- **Relevance**: Combo discovery through experimentation, multi-button chords, pressure under enemy swarms.

### **Arx Fatalis**
- **Input Model**: Draw rune symbols with mouse in 3D space, combine into spell formulas.
- **Complexity**: 20+ runes memorization, multi-rune sequences, drawing accuracy under time pressure.
- **Forgiveness**: Pause during drawing (no combat pressure while casting), forgiving shape recognition.
- **Relevance**: Gestural input, spell language literacy, spatial mouse control.

### **Lichdom: Battlemage**
- **Input Model**: Left/right mouse + modifiers (sigils) for different spell effects.
- **Complexity**: Sigil switching, combo timing, cooldown management, synergy crafting.
- **Forgiveness**: Shield absorbs damage, cooldown overlap allows constant casting.
- **Relevance**: Dual-input spell variations, real-time elemental switching.

### **In Verbis Virtus**
- **Input Model**: Speak spell words into microphone in fictional language.
- **Complexity**: Pronunciation accuracy, memorizing incantations, enunciation under pressure.
- **Forgiveness**: Forgiving voice recognition, no combat time pressure during casting.
- **Relevance**: Unique input modality (voice), memorization challenge, immersion through performance.

### **Dark Messiah of Might and Magic**
- **Input Model**: Hotkey spells with aim/timing requirements (freeze ground, telekinesis push).
- **Complexity**: Environmental combos (kick into spikes, oil + fire), mana management, physics timing.
- **Forgiveness**: Generous hitboxes, mana potions, melee fallback.
- **Relevance**: Physics-based spells, environmental interaction, aim precision.

### **Spellbreak**
- **Input Model**: Dual-wield gauntlets (left/right click), aim projectiles, dodge roll.
- **Complexity**: Gauntlet combos (toxin cloud + fire = explosion), projectile leading, aerial control.
- **Forgiveness**: Large AoE spells, generous hitboxes, respawns in Battle Royale.
- **Relevance**: Third-person aim, elemental combos, movement tech (hovering, teleports).

### **Waltz of the Wizard (VR)**
- **Input Model**: Grab ingredients, mix in cauldron, perform hand gestures with motion controllers.
- **Complexity**: Spatial accuracy, gesture recognition, timing pouring/mixing.
- **Forgiveness**: No fail states; sandbox experimentation focus.
- **Relevance**: Physical gesture input, tactile interaction, spatial choreography.

### **Portal / Portal 2**
- **Input Model**: Aim portal gun, shoot portals, momentum-based platforming.
- **Complexity**: Flinging (portal + gravity), mid-air portal placement, timing hazards.
- **Forgiveness**: No death penalty (respawn at checkpoint), unlimited portal attempts.
- **Relevance**: Spatial reasoning + timing, cursor precision under momentum, test chamber iteration.

### **Braid**
- **Input Model**: Platforming + time rewind (hold Shift).
- **Complexity**: Time-immune objects, world-specific mechanics (different time rules per world).
- **Forgiveness**: Time rewind negates mistakes; puzzle solving is the gate, not execution.
- **Relevance**: Time manipulation as input modifier, forgiving execution baseline.

### **The Talos Principle**
- **Input Model**: Pick up, place, activate objects (jammers, connectors, boxes).
- **Complexity**: Laser routing, timing pressure plates, sequence optimization.
- **Forgiveness**: No time pressure, unlimited attempts, hint system.
- **Relevance**: Spatial manipulation puzzles, minimal execution pressure (purely cognitive).

### **Fez**
- **Input Model**: Rotate world 90° (shoulder buttons), platforming with 2D/3D perspective shifts.
- **Complexity**: Hidden rooms, QR codes, cryptography, rotation timing mid-jump.

- **Relevance**: Perspective manipulation, timing jumps with rotation, cognitive + spatial challenge.

### **SpaceChem / Opus Magnum (Zachtronics)**

### **Stephen's Sausage Roll**

### **Baba Is You**
- **Input Model**: Move Baba, push word blocks to rewrite rules.
- **Complexity**: Rule parsing, emergent interactions, sequence dependencies.
- **Forgiveness**: Unlimited undo, no time pressure; purely logic-based.
- **Relevance**: Linguistic puzzle solving, minimal execution demands (movement only).

### **Antichamber**

## Chord-Based / Simultaneous Input Games

### **Thumper**
- **Input Model**: Press button on beat, hold for lanes, turn + press for corners.
- **Complexity**: Boss sequences (multi-step chord patterns), simultaneous turn + press, increasing speed.
- **Forgiveness**: Checkpoints per segment, Practice Mode, Play+ for mastery.
- **Relevance**: Rhythm + spatial input, chord-like turns (directional + button), escalating pressure.

### **Furi**
- **Input Model**: Parry (Y), dodge (B), shoot (RT), charge shot (hold RT), slash (X).
- **Complexity**: Parry timing (frames), charge shot + dodge (simultaneous), pattern phases.
- **Forgiveness**: Health bars, phase checkpoints, generous parry window (~10 frames).
- **Relevance**: Multi-button simultaneous inputs (charge + dodge), rhythm-based boss patterns.

### **One Finger Death Punch 1 & 2**
- **Input Model**: Two buttons (left/right) pressed in rhythm to counter attacks.
- **Complexity**: Brawler rounds (rapid sequences), bomb timers, multi-phase mini-bosses.
- **Forgiveness**: Generous timing windows, slow-motion on critical hits, skills auto-unlock.
- **Relevance**: Binary chord-like input (left+right not simultaneous, but sequenced), rhythm combat.

### **A Dance of Fire and Ice**
- **Input Model**: Press button on every beat to guide dual orbiting planets along track.
- **Complexity**: Polyrhythms, tempo changes, misdirection (visual vs audio cues).
- **Forgiveness**: Checkpoints per world, calibration tool, practice mode.
- **Relevance**: Strict 1:1 rhythm (every beat = input), no chord, but absolute timing precision.

### **Sayonara Wild Hearts**
- **Input Model**: Steer (analog), dodge (button), rhythmic quick-time events (QTEs).
- **Complexity**: Lane switching + rhythm, boss QTE chords, speed changes.

- **Relevance**: Steering + button timing fusion, QTE chords under speed pressure.

### **Clone Hero / Phase Shift**

- **Complexity**: Chord shapes, fast strumming, hammer-ons, sustains.

### **Ape Out**

- **Complexity**: Directional throws under jazz-synced rhythm, timing door breaks, crowd control.
- **Forgiveness**: Procedural levels, generous throw hitboxes, quick respawn.
- **Relevance**: Rhythm + directional precision, momentum-based input (analog + button combo).

### **Before the Echo (Sequence)**

## Extended Takeaways for Spellcasting Design

### **From Spellcasting Games:**
- **Magicka combo logic**: Element interactions create emergent complexity; players experiment to discover.
- **Arx Fatalis gestures**: Spatial input (mouse drawing) adds physicality; memorization becomes skill expression.
- **Lichdom dual-input**: Left/right mouse for different spell aspects (element + modifier) = fast switching.
- **In Verbis Virtus voice**: Alternative input modalities (voice, gesture) increase immersion but require forgiving recognition.
- **Spellbreak combos**: Environmental interactions (element + element) reward creative combination.

### **From Puzzle Games:**
- **Portal execution forgiveness**: Let spatial/cognitive challenge dominate; execution should be accessible.
- **Zachtronics optimization**: Make the puzzle **designing the inputs**, not executing them under pressure.
- **Braid time manipulation**: Forgiving mechanics (undo, rewind) let players iterate without punishment.
- **Fez perspective shifts**: Modifier inputs (rotate view) change problem space, not core input complexity.

### **From Chord Games:**
- **Thumper rhythm chords**: Directional + button as unified "chord"; timing is the challenge, not memorizing shape.
- **Furi simultaneous actions**: Charge shot + dodge = multi-tasking under pressure; accessible individually, challenging combined.
- **Clone Hero chord shapes**: Visual representation (note highway) shows chord shape before execution; readable telegraph.
- **Before the Echo multi-tasking**: Rhythm execution competes with tactical decision-making for attention.

### **New Design Principles:**
1. **Chord readability**: Show simultaneous inputs visually (all buttons at once) vs sequential (one after another).
2. **Combo discovery**: Reward experimentation; let players find interactions through play, not wikis.
3. **Gestural input**: Physical motions (mouse gestures, VR hands) increase immersion but need forgiving recognition.
4. **Multi-tasking tension**: Splitting attention (execute spell + dodge enemy) creates pressure without complex inputs.
5. **Environmental combos**: Spell + world interactions (freeze water, ignite oil) add tactical depth without input complexity.
6. **Input as puzzle**: Make designing the input sequence the challenge (Zachtronics), not executing it under time pressure.
7. **Modifier inputs**: Stance/mode switches change spell behavior without increasing base complexity (Nioh-style).
